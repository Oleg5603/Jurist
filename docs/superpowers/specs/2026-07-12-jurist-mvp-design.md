# Jurist — MVP веб-агента для частного юриста

## Назначение
Веб-приложение для одного частного юриста (право РФ): чат по правовым вопросам,
генерация типовых документов по вводным, анализ загруженного договора на риски.
Не заменяет юридическую консультацию — все результаты требуют проверки юристом.

## Стек
- Backend: Python, FastAPI
- Frontend: статичный HTML/JS/CSS, без сборки (без Node/npm)
- LLM: OpenRouter (единый ключ на все модели, как в `veterans_agents/veteran_bot.py`)
- Хранилище: JSON/файлы на диске, без БД
- Auth: один логин/пароль на всех, cookie-сессия

```
Jurist/

├── storage.py                — чтение/запись JSON
├── static/

```

## Функционал

### 1. Вход
- Единая форма логин/пароль (`/login`), сверка с `APP_LOGIN` / `APP_PASSWORD` из `.env`.
- При успехе — подписанная cookie-сессия, дальнейшие запросы к `/api/*` её проверяют.
- Без ролей и многопользовательской модели — сознательное упрощение MVP.

### 2. Чат по правовым вопросам
- `POST /api/chat` — принимает сообщение + id сессии чата.
- Системный промпт: юридический ассистент, ориентированный на право РФ,
  явно указывает, что ответ не является юридической консультацией.
- История хранится в `data/chats/<session>.json`, подгружается при следующих
  сообщениях той же сессии (аналогично `user_model`/истории в veteran_bot).
- Ответ идёт через OpenRouter, модель по умолчанию — бесплатная (Llama),
  с возможностью переключить на платную в интерфейсе (как в veteran_bot).

### 3. Генерация документов
- Форма с полями: тип документа (NDA / договор / претензия / исковое заявление),
  стороны, предмет, суммы, доп. условия (свободный текст).
- `POST /api/documents/generate` — собирает промпт с инструкцией и вводными,
  получает текст документа от LLM.
- Результат сохраняется в `data/documents/<id>.json` (вводные + текст + дата),
  отдаётся пользователю как .txt для скачивания.
- История сгенерированных документов доступна в интерфейсе (список с датами).

### 4. Анализ договора
- Загрузка файла (.txt/.docx/.pdf) через `POST /api/contracts/analyze`.
- Извлечение текста (docx/pdf парсится в текст стандартными библиотеками).
- Промпт: найти риски с маркировкой 🔴 критический / 🟡 важный / 🟢 приемлемый,
  указать пропущенные типовые защитные пункты, предложить конкретные правки.
- Исходный файл и результат анализа сохраняются в `data/contracts/<id>/`.
- Результат отображается в интерфейсе структурированно (список рисков с цветовой меткой).

## Обработка ошибок
Единообразный try/except вокруг вызовов OpenRouter (таймаут, сетевая ошибка,
невалидный ответ) — пользователю показывается понятное сообщение об ошибке,
ошибка не проглатывается молча и не выглядит как штатный ответ ассистента.

## Тестирование
Юнит-тесты не пишутся — это одноразовое приложение для одного пользователя.
Проверка — ручной прогон трёх сценариев (чат, генерация документа, анализ
договора) после реализации, через `/verify`.

## Границы MVP (сознательно не делаем)
- Мониторинг новостей/изменений законодательства
- Поиск дел в VDR/файловых хранилищах
- Compliance-проверки (GDPR/CCPA и т.п. — не актуально для РФ)
- Мульти-агентная оркестрация, RAG по базе законов
- Многопользовательская модель с ролями

Эти пункты не исключаются навсегда — при появлении конкретного запроса
добавляются отдельным циклом спецификации.
