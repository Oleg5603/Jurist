## Что это
Веб-приложение для частного юриста (РФ). MVP из трёх функций, без мульти-агентного оркестратора (сознательно урезано от 9-агентной идеи — слишком долго, заточено под США).

## Архитектура (зафиксировано)
Один Python-процесс: FastAPI backend + статичный HTML/JS фронт (без Node/сборки). Один общий логин/пароль на всех (cookie-сессия). Хранение — JSON-файлы на диске, без БД (по аналогии с sibvaleo/veteran_bot). LLM — через **OpenRouter** (не Anthropic напрямую — у пользователя личная подписка Claude, а не оплаченный API-баланс, поэтому прямые вызовы Anthropic API упирались в `credit balance too low`).

## Функционал MVP (зафиксировано, см. спеку)
1. **Вход** — один логин/пароль (`APP_LOGIN`/`APP_PASSWORD` в `.env`), cookie-сессия.
2. **Чат по правовым вопросам** — юрист пишет вопрос, ответ через OpenRouter, промпт под право РФ с явной пометкой «не юридическая консультация», история сохраняется в `data/chats/`.
3. **Генерация документов** — форма (тип: NDA / договор / претензия / исковое, стороны, предмет, суммы) → черновик от LLM → скачивание, сохраняется в `data/documents/`.
4. **Анализ договора** — загрузка файла (.txt/.docx/.pdf) → риски 🔴🟡🟢, пропущенные пункты, конкретные правки → сохраняется в `data/contracts/<id>/`.

**Сознательно НЕ делаем в MVP** (пользователь подтвердил 2026-07-12 16:53): мониторинг законов/новостей, поиск дел в VDR/хранилищах, compliance по GDPR/CCPA, мульти-агентная оркестрация. Пользователь просил **напоминать об этом решении не чаще раза в день**, если тема всплывёт снова — не давить, просто не забывать про урезанный scope.
